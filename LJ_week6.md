## Content Insights:
This week we learned about project management. 
Student projects can fail for a few common reasons: Tasks are too large or vague, work is not committed regularly, communication is inconsistent, integration happens too late. 
Below is an example workflow
Create a task on group board -> Assign someone to the task -> Create a branch and begin work -> Commit and push regularly -> open a pull request, review, merge -> Move task to done
Make branches based on task
Tasks should be small and manageable. 
Prioritize communication and clarity.
